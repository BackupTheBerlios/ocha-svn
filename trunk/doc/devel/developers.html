Checking out and compiling ocha

This document quickly describes how to checkout ocha from subversion and compile it. It is meant for developers. Most people will be happy downloading the distribution and following the instructions given in the documentation or on the homepage

Why would you want to compile ocha straight out of subversion ? Maybe because you would like to fix it. Maybe you would like to get access to the latest version with all the unstable features. Maybe you just like danger... Whatever the reason, here's what you should do:

1. Checkout out the latest version

A Subversion HOWTO is available on berlios.de. I won't get into details here, just give the command you need to type to checkout the main version of ocha:

svn checkout svn://svn.berlios.de/ocha/trunk

Warning: the version of ocha on subversion may not work at all, may not do anything useful or may not even compile at all. Don't be surprised if something goes wrong; just report it on the developers forum

2. Make sure you've got everything

In addition to everything you need to compile and run ocha that's listed in the documentation, you'll need the following software to build ocha straight out of subversion:

  • gnome-autogen.sh
  • autoconf >= 2.53
  • automake >= 1.6
  • libtool >= 1.5
  • pkg-config >= 0.14.0

The most important of all these and the hardest to find is probably gnome-autogen.sh. It's the script that does all the work of generating the configure script you'll need to compile the application.

You will find gnome-autogen.sh in a package called gnome-common in most distribution (at least Debian and Fedora.) GNOME distribute it under this name as well.

3. Generate the configure script

Go into the ocha's trunk directory and run ./autogen.sh. It'll generate configure and run it for the first time.

Once this is done, you can edit the source files and run make to compile it, as usual.

4. Run the test cases (make check)

Depending on what you're working on, you might want to run the test cases using make check. This won't work unless you've got the library check version 0.9.2 or more installed on your system.

Download it from check.sourceforge.net. It is also available in a Debian package called check.

5. Generate the API documentation (make doxygen)

The API documentation might help you find your way into the code. To generate it, first install doxygen. Then go into the directory src and type make doxygen. You'll then find the generated API documentation in HTML in doc/devel/api/html.
